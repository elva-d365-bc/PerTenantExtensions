    Usage Scenarios

    Scenario 1: Creating a New Contract and Adding Services

    Goal: Create a new contract with a customer for premises rental services.

    Participants: Contract Manager, Accountant

    Prerequisites:

    • Customer is registered in the system
    • Services are created in the service catalog
    • Objects (premises) are registered

    Steps:

    1. Open Contract List

      • Search for "ELVA BLS Contracts" or "Contracts"

    2. Create New Contract

      • Press "New"
      • Enter or let the system generate the contract number
      • Enter contract date and approval date

    3. Specify Contract Partner

      • Choose partner source: Customer
      • Enter partner number
      • System automatically fills in partner name and other information

    4. Set Contract Validity Period

      • Enter "From Date"
      • Enter "To Date" or leave blank for indefinite contract

    5. Add Contract Services

      • Go to "Contract Services" subpage
      • Add new line
      • Choose service code
      • Choose object (if needed)
      • Specify service period
      • Set quantity and price
      • Specify calculation period

    6. Activate Contract

      • Verify all data
      • Change status to "Active"

    Result: Contract is created and ready for calculation.

    Scenario 2: Using Calculation Worksheet for Service Calculation

    Goal: Calculate all active contract services for the specified period.

    Participants: Accountant

    Prerequisites:

    • Active contracts with services created
    • Calculation worksheet template defined
    • Periods defined

    Steps:

    1. Open Calculation Worksheet

      • Search for "ELVA BLS Calculation Worksheet"

    2. Select or Create Worksheet Batch

      • Choose worksheet template
      • Select or create batch

    3. Generate Calculation Lines

      • Execute "Generate Calculation Lines" function
      • Specify period filters (from date - to date)
      • Choose calculation period code (if needed)
      • Specify other filters (contract, service, object)
      • Confirm generation

    4. Review Generated Lines

      • Check created calculation lines
      • Verify quantities and prices are correct
      • Make necessary manual corrections

    5. Calculate Prices and Discounts

      • System automatically calculates prices from price lists
      • System applies discounts from contract or service discounts

    6. Transfer to Calculation Entries

      • Execute "Post Calculation Lines" function
      • Calculations are transferred to calculation entry table
      • Worksheet lines are deleted or marked as processed

    Result: Services are calculated and prepared for journal posting.

    Scenario 3: Posting Calculations in Journal

    Goal: Transfer calculated services to billing journal and post.

    Participants: Accountant

    Prerequisites:

    • Calculated services in calculation worksheet or calculation entries

    Steps:

    1. Open Billing Journal

      • Search for "ELVA BLS Journal"

    2. Select Journal Template and Batch

      • Choose appropriate journal template
      • Select or create batch

    3. Transfer Calculations from Worksheet

      • Execute "Transfer from Calculation Entries" function
      • Specify filters (period, partner, contract)
      • Confirm transfer

    4. Check Journal Lines

      • Review created journal lines
      • Check partners, services, amounts
      • Make corrections if needed

    5. Execute Journal Check

      • Execute "Test Report" function
      • Verify there are no errors

    6. Post Journal

      • Execute "Post" function
      • Choose posting method:
        • Only post in billing entries
        • Create sales/purchase invoices
        • Prepare export to external system

    7. Check Billing Entries

      • Open "Billing Ledger Entries"
      • Verify created entries

    Result: Services are posted and invoices created or prepared for further processing.

    Scenario 4: Object Registration and Linking to Contract

    Goal: Register new object (e.g., apartment) and link it to rental contract.

    Participants: Object Manager, Contract Manager

    Prerequisites:

    • Object categories defined
    • Customer is registered in the system

    Steps:

    1. Open Object List

      • Search for "ELVA BLS Objects"

    2. Create New Object

      • Press "New"
      • Enter or let generate object code
      • Enter object name (e.g., "Apartment No. 12")

    3. Specify Object Category

      • Choose object category (e.g., "Apartments")

    4. Enter Object Parameters

      • Enter object parameters (total area, living area, etc.)
      • These parameters can be used in quantity calculation

    5. Enter Address and Contact Information

      • Enter address
      • Enter contact information

    6. Add Related Partners

      • Go to "Related Partners" subpage
      • Add owner or tenant

    7. Link Object to Contract

      • Open contract
      • Go to "Contract Services"
      • Choose object code in service line

    Result: Object is registered and linked to contract, now services can be calculated for this object.

    Scenario 5: Period Recalculation and Corrections

    Goal: Recalculate previous period calculations if errors or changes are found.

    Participants: Accountant

    Prerequisites:

    • Previous period calculations are done
    • Errors or changes identified

    Steps:

    1. Identify Necessary Changes

      • Review billing entries
      • Identify errors or changes

    2. Create Corrections in Journal

      • Open billing journal
      • Create new correction lines
      • Specify negative amounts to cancel erroneous entries
      • Add new lines with correct amounts

    3. Or Repeat in Calculation Worksheet

      • Open calculation worksheet
      • Generate calculation lines with "Recalculate" option
      • Specify period to recalculate
      • System will create correction lines

    4. Post Corrections

      • Post journal with corrections
      • Check billing entries

    Result: Previous period calculations are corrected, correction entries created.
