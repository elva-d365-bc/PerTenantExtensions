    Main Modules

    1. Contracts

    The contract module is the core element of the system, providing:

    • Contract registration with numbering
    • Contract status management (Active, Closed, Interrupted, Inactive)
    • Partner (customer/vendor) linking
    • Contract service definition
    • Contract document attachment
    • Discount setup

    Main contract data groups:

    • Basic Information: contract number, date, approval date, external contract number
    • Contract Partners: supplier/customer, bill-to recipient, contact persons
    • Contract Period: valid from date, valid to date, actual closing date
    • Financial Information: base amount, amount incl. VAT, currency
    • Responsible Persons: responsible person, responsible accountant
    • Contract Services: list of services with periods, quantities, and prices

    2. Services

    The service catalog defines what can be included in contracts:

    • Service Code and Description - unique identifier and description
    • Service Groups - logical grouping
    • Service Variants - service variations (e.g., different volumes, quality levels)
    • Unit of Measure - quantity unit of measure
    • Quantity Source - how quantity is determined (manually, from calendar, from object value)
    • Pricing - default price, price coefficients
    • Automatic Dimension Assignment - accounting dimensions

    3. Objects

    The object module allows registration of physical or logical objects for which services are provided:

    • Object Code and Name
    • Object Category - object classification
    • Object Parameters - numeric parameters (total area, living area, etc.)
    • Object Address and Contact Information
    • Related Partners - owners, tenants, etc.
    • Dimensions - accounting dimension values

    4. Calculation Worksheet

    The calculation worksheet is the main tool for service calculation:

    • Worksheet Templates - different types of calculation templates
    • Worksheet Batches - work with specific calculations
    • Calculation Line Generation - automatic line creation from contracts
    • Price and Discount Calculation - pricing based on price lists and discount application
    • Calculation Review and Correction - manual editing
    • Transfer to Journal - sending prepared calculations for posting

    5. Journals

    Journals are used for posting approved calculations:

    • Journal Templates - different journal types
    • Journal Batches - work sessions for specific calculations
    • Journal Lines - individual entries
    • Recurring Journals - automation of repeated entries
    • Posting - conversion of billing entries to invoices or preparation for export

    6. Periods

    Period definitions determine calculation periodicity:

    • Period Code and Description
    • Period Start - month/quarter/year start or end
    • Period Starting Date Formula - automatic calculation
    • Period Length Formula - month, quarter, year, etc.
    • Recalculate Previous Period - ability to recalculate previous period
