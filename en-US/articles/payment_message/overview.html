    Overview

    Payment Message is a customization integrated with Dynamics 365 Business Central developed by ELVA.

    Payment Message is an adjustment that automates the filling of the Message to the recipient in the Payment log even in cases where the payment is applied to more than one document.

    Description

    The function will process the filtered records of the user whose Account type is Supplier and fill the Message to Recipient field with the text Invoice. A, B, C, where the External document no. Entries that do not qualify will be skipped.

    If the Message to the recipient exceeds the maximum field length (140 characters), it will be truncated.

    Payment journal entries will be controlled by the value in the Exported to payment file field. If the value is Yes, a warning will be displayed. If there are multiple rows with such a tag, one warning will be displayed. If the user approves it, the Message to Recipient field will be filled in as described above. If not, the existing value will be left in the field.

    Benefits

    By automatically filling the field Message to the recipient with the external document No. of the assigned supplier book record, the time required for the preparation of payment orders is reduced and the possibility of errors by manually specifying incorrect documents is reduced.

    Assumptions and limitations

    Only supplier book entries will be processed, for payments to customers and employees, the Report must be filled in manually by the recipient.
